### Ayak 프로젝트

# 1. 프로젝트 소개

AYAK(아약)은 사용자가 현재 복용 중인 약을 입력하면  
해당 의약품의 주의사항, 복용 정보, 부작용 등을 쉽고 직관적으로 확인할 수 있도록 도와주는 간단한 웹 서비스입니다.

기존 의약품 정보는 설명이 길고 어려운 경우가 많기 때문에,  
AYAK은 사용자가 꼭 알아야 할 핵심 정보만 간단하게 보여주는 것을 목표로 합니다.

---

# 2. 프로젝트 목적

- 일반 사용자가 의약품 정보를 쉽게 확인할 수 있도록 도움
- 복잡한 의약품 설명서를 간단한 UI로 제공
- 공공데이터를 활용한 실생활형 웹 서비스 구현
- 사용자의 안전한 복용 습관 형성 지원

---

# 3. 주요 기능

## 1) 의약품 검색

사용자가 약 이름을 검색하면 관련 의약품 정보를 조회할 수 있습니다.

예시:

- 타이레놀
- 판콜
- 게보린
- 지르텍

---

## 2) 주의사항 안내

검색한 의약품의 주요 주의사항을 카드 형태로 제공합니다.

예시:

```txt
음주 전후 복용 주의
졸음 유발 가능
식후 복용 권장
운전 전 복용 주의
```

---

## 3) 효능 및 복용 정보 제공

- 효능
- 사용 방법
- 복용 시 주의사항
- 부작용

등의 핵심 정보를 제공합니다.

---

## 4) 복용 중인 약 등록 (선택 기능)

사용자가 현재 복용 중인 약을 등록하여  
나만의 복용 리스트를 관리할 수 있습니다.

---

# 4. 기대 효과

- 의약품 정보를 쉽고 빠르게 확인 가능
- 사용자 친화적인 UI를 통한 접근성 향상
- 공공데이터 활용 경험 및 데이터 정제 경험 확보
- 실생활 문제를 해결하는 서비스 경험 제공

---

## Backend / Database

- Supabase (PostgreSQL)

# 6. 데이터 활용 방식

AYAK은 공공 의약품 데이터를 기반으로  
필요한 데이터를 정제하여 자체 DB에 저장하고 서비스에 활용합니다.

사용 데이터:

- 식품의약품안전처\_의약품개요정보(e약은요)

활용 정보:

- 제품명
- 효능
- 사용법
- 복용 시 주의사항
- 부작용

---

# 7. 서비스 흐름

```txt
사용자 검색
    ↓
의약품 조회
    ↓
DB에서 정보 검색
    ↓
주의사항 및 복용 정보 출력
```

---

# 8. 차별점

기존 의약품 정보 서비스는 설명이 길고 복잡한 경우가 많습니다.

AYAK은:

- 꼭 필요한 정보만 간단하게 제공
- 직관적인 카드 UI 사용
- 사용자 친화적인 디자인 구성

을 통해 쉽고 빠른 정보 확인 경험을 제공합니다.

# 2. 의약품 검색 기능

## 2.1 의약품 검색

- [x] 사용자는 약 이름을 입력하여 의약품을 검색할 수 있다.
- [x] 검색 결과는 약 이름 기준으로 조회된다.
- [x] 검색 결과가 없을 경우 안내 문구를 표시한다.

## 2.2 의약품 상세 조회

- [x] 사용자는 검색 결과에서 특정 의약품을 선택하여 상세 정보를 확인할 수 있다.
- [x] 상세 정보에는 효능, 사용 방법, 주의사항, 부작용 정보가 포함된다.

# 3. 내 복용약 기능

## 3.1 내 복용약 목록 조회

- [x] 로그인한 사용자는 내 복용약 페이지에서 등록한 복용 약품 목록을 조회할 수 있다.
- [x] 등록된 약품의 주요 주의사항 요약 정보를 확인할 수 있다.

## 3.2 복용 약품 등록

- [x] 로그인한 사용자는 의약품 상세 페이지에서 해당 약품을 자신의 복용 약품 목록에 등록할 수 있다.
- [x] 동일한 약품은 중복 등록되지 않도록 한다.

## 3.3 복용 약품 삭제

- [x] 로그인한 사용자는 내 복용약 페이지에서 등록한 약품을 삭제할 수 있다.

## 3.4 주의사항 요약 표시

- [x] 내 복용약 페이지에서는 각 약품별 핵심 주의사항을 카드 형태로 표시한다.

예시:

```

- 과다복용 주의

판콜
- 졸음 유발 가능
- 운전 전 복용 주의
```

# 프로젝트 회고

이번 프로젝트를 진행하며 하네스 엔지니어링 방식의 개발을 직접 경험해볼 수 있었습니다.

프로젝트를 진행하면서 느낀 가장 큰 점은, 기존에 얕게나마 학습해두었던 React와 웹 개발 지식이 실제로
AI와 협업할 때 큰 도움이 되었다는 점입니다.

예를 들어 어떤 페이지의 어떤 부분을 수정하고 싶은지 비교적 구체적으로 설명할 수 있었고,
그 과정에서 AI가 어떤 방식으로 구조를 잡고 코드를 생성하는지 흐름을 관찰할 수 있었습니다.

또한 API 연동 방법, 데이터베이스 적재 방식, 데이터 호출 구조 등을 질문하고 대화하며
실제 프로젝트에 반영할 수 있었던 경험이 인상 깊었습니다.

---

반대로 생각해보면, 만약 기반 지식 없이 단순히 바이브 코딩 방식으로 접근했다면
프롬프트를 어떻게 입력해야 하는지조차 막막했을 것이라고 느꼈습니다.

AI가 생성한 결과물이 왜 그렇게 동작하는지 이해하지 못한다면
수정이나 확장도 어려웠을 것이고, 원하는 방향으로 프로젝트를 이끌어가기 힘들었을 것 같습니다.

---

또 하나 크게 느낀 점은, 단단한 기획과 명확한 의도를 기반으로 설계하고
이를 실제 배포까지 안정적으로 완성하는 과정이 생각보다 훨씬 어렵다는 점이었습니다.

개발을 진행하는 과정에서 중간중간 핵심 기능이나 초기 기획 의도를 놓치는 실수도 있었고,
기능이 추가될수록 프로젝트의 방향성이 흐려질 수 있다는 것도 체감했습니다.

특히 AI에게 개발의 주도권을 온전히 맡기게 되면,
프로젝트가 진행될수록 처음 의도했던 서비스의 핵심 가치가 흐려질 수 있다는 점을 느꼈습니다.

결국 AI는 매우 강력한 도구이지만,
프로젝트의 방향성과 우선순위를 결정하는 것은 개발자의 역할이라는 점을 다시 한 번 깨달았습니다.

---

이번 프로젝트를 통해 AI와 협업하는 개발 방식의 가능성을 경험할 수 있었고,
앞으로는 더욱 깊은 학습과 기술 이해를 바탕으로 AI와의 프로젝트에서도
개발자로서 주도권을 가지고 설계와 구현을 이끌어갈 수 있어야겠다고 생각했습니다.
