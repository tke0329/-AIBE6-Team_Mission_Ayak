### Ayak 프로젝트

# 1. 프로젝트 소개

AYAK(아약)은 사용자가 현재 복용 중인 약을 입력하면  
해당 의약품의 주의사항, 복용 정보, 부작용 등을 쉽고 직관적으로 확인할 수 있도록 도와주는 간단한 웹 서비스입니다.

기존 의약품 정보는 설명이 길고 어려운 경우가 많기 때문에,  
AYAK은 사용자가 꼭 알아야 할 핵심 정보만 간단하게 보여주는 것을 목표로 합니다.

---

# 2. 프로젝트 목적

- 일반 사용자가 의약품 정보를 쉽게 확인할 수 있도록 도움
- 복잡한 의약품 설명서를 간단한 UI로 제공
- 공공데이터를 활용한 실생활형 웹 서비스 구현
- 사용자의 안전한 복용 습관 형성 지원

---

# 3. 주요 기능

## 1) 의약품 검색

사용자가 약 이름을 검색하면 관련 의약품 정보를 조회할 수 있습니다.

예시:

- 타이레놀
- 판콜
- 게보린
- 지르텍

---

## 2) 주의사항 안내

검색한 의약품의 주요 주의사항을 카드 형태로 제공합니다.

예시:

```txt
음주 전후 복용 주의
졸음 유발 가능
식후 복용 권장
운전 전 복용 주의
```

---

## 3) 효능 및 복용 정보 제공

- 효능
- 사용 방법
- 복용 시 주의사항
- 부작용

등의 핵심 정보를 제공합니다.

---

## 4) 복용 중인 약 등록 (선택 기능)

사용자가 현재 복용 중인 약을 등록하여  
나만의 복용 리스트를 관리할 수 있습니다.

---

# 4. 기대 효과

- 의약품 정보를 쉽고 빠르게 확인 가능
- 사용자 친화적인 UI를 통한 접근성 향상
- 공공데이터 활용 경험 및 데이터 정제 경험 확보
- 실생활 문제를 해결하는 서비스 경험 제공

---

# 5. 사용 기술

## Frontend

- React / Next.js
- Tailwind CSS

## Backend / Database

- Supabase (PostgreSQL)

## Data

- 식품의약품안전처 의약품 공공데이터

---

# 6. 데이터 활용 방식

AYAK은 공공 의약품 데이터를 기반으로  
필요한 데이터를 정제하여 자체 DB에 저장하고 서비스에 활용합니다.

사용 데이터:

- 식품의약품안전처\_의약품개요정보(e약은요)

활용 정보:

- 제품명
- 효능
- 사용법
- 복용 시 주의사항
- 부작용

---

# 7. 서비스 흐름

```txt
사용자 검색
    ↓
의약품 조회
    ↓
DB에서 정보 검색
    ↓
주의사항 및 복용 정보 출력
```

---

# 8. 차별점

기존 의약품 정보 서비스는 설명이 길고 복잡한 경우가 많습니다.

AYAK은:

- 꼭 필요한 정보만 간단하게 제공
- 직관적인 카드 UI 사용
- 사용자 친화적인 디자인 구성

을 통해 쉽고 빠른 정보 확인 경험을 제공합니다.

---

# 9. 향후 확장 가능성

- 약 조합 위험 안내
- AI 기반 복용 주의사항 요약
- 즐겨찾기 기능
- 복용 알림 기능
- 모바일 앱 확장

---

# 10. 한 줄 소개

# AYAK 요구사항 명세서

---

# 1. 회원 기능

## 1.1 회원가입

- [x] 사용자는 이메일과 비밀번호를 입력하여 회원가입할 수 있다.
- [x] 회원가입 시 이메일은 중복될 수 없다.
- [x] 비밀번호는 필수 입력값이다.

## 1.2 로그인

- [x] 사용자는 가입한 이메일과 비밀번호로 로그인할 수 있다.
- [x] 로그인 성공 시 서비스를 이용할 수 있다.

## 1.3 로그아웃

- [x] 로그인한 사용자는 로그아웃할 수 있다.
- [x] 로그아웃 후에는 내 복용약 페이지 접근이 제한된다.

## 1.4 마이페이지

- [x] 사용자는 자신의 기본 회원 정보를 확인할 수 있다.
- [x] 사용자는 로그아웃할 수 있다.

---

# 2. 의약품 검색 기능

## 2.1 의약품 검색

- [x] 사용자는 약 이름을 입력하여 의약품을 검색할 수 있다.
- [x] 검색 결과는 약 이름 기준으로 조회된다.
- [x] 검색 결과가 없을 경우 안내 문구를 표시한다.

## 2.2 의약품 상세 조회

- [x] 사용자는 검색 결과에서 특정 의약품을 선택하여 상세 정보를 확인할 수 있다.
- [x] 상세 정보에는 효능, 사용 방법, 주의사항, 부작용 정보가 포함된다.

---

# 3. 내 복용약 기능

## 3.1 내 복용약 목록 조회

- [x] 로그인한 사용자는 내 복용약 페이지에서 등록한 복용 약품 목록을 조회할 수 있다.
- [x] 등록된 약품의 주요 주의사항 요약 정보를 확인할 수 있다.

## 3.2 복용 약품 등록

- [x] 로그인한 사용자는 의약품 상세 페이지에서 해당 약품을 자신의 복용 약품 목록에 등록할 수 있다.
- [x] 동일한 약품은 중복 등록되지 않도록 한다.

## 3.3 복용 약품 삭제

- [x] 로그인한 사용자는 내 복용약 페이지에서 등록한 약품을 삭제할 수 있다.

## 3.4 주의사항 요약 표시

- [x] 내 복용약 페이지에서는 각 약품별 핵심 주의사항을 카드 형태로 표시한다.

예시:

```
타이레놀
- 음주 전후 복용 주의
- 과다복용 주의

판콜
- 졸음 유발 가능
- 운전 전 복용 주의
```

---

# 4. 주의사항 안내 기능

## 4.1 주의사항 표시

- [x] 의약품 상세 페이지에서 해당 약품의 주의사항을 표시한다.
- [x] 내 복용약 페이지에서는 등록된 약품의 핵심 주의사항을 요약하여 표시한다.

## 4.2 부작용 정보 표시

- [x] 의약품 상세 페이지에서 주요 부작용 정보를 제공한다.

## 4.3 안내 문구 제공

- [x] 서비스 하단에 의료 전문가의 진단을 대체하지 않는다는 안내 문구를 표시한다.

---

# 5. 데이터 관리

## 5.1 의약품 데이터 저장

- [x] 식품의약품안전처 의약품개요정보(e약은요) 데이터를 기반으로 필요한 정보를 정제하여 자체 DB에 저장한다.

## 5.2 저장 데이터 항목

- [x] 제품명
- [x] 효능
- [x] 사용 방법
- [x] 주의사항
- [x] 부작용
- [x] 보관 방법

---

## 6.1 메인 페이지

- [x] 서비스 소개 문구를 표시한다.

## 6.2 검색 결과 페이지

- [x] 검색된 의약품 목록을 표시한다.
- [x] 사용자는 목록에서 상세 페이지로 이동할 수 있다.

## 6.3 의약품 상세 페이지

## 6.4 내 복용약 페이지

## 6.5 마이페이지

# 7. 비기능 요구사항

## 7.1 사용성

- [x] 사용자는 약 이름을 쉽게 검색할 수 있어야 한다.
- [x] 복잡한 의약품 정보를 간단하고 직관적인 형태로 제공해야 한다.

- [x] 제공되는 정보는 참고용이며 의료 전문가의 진단을 대체하지 않는다는 안내 문구를 제공한다.
