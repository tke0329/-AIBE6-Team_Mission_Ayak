# CONTEXT

- AYAK 1차 MVP 화면 구현이 완료되었다.
- 현재 결과물은 목업 기반 화면, Supabase `medications` 기반 검색/상세/홈 추천, Supabase Auth 기반 로그인/회원가입/로그아웃, 보호 라우트, `user_medications` 기반 내 복용약 관리까지 포함한다.
- 공개 페이지 지연을 줄이기 위해 클라이언트 인증 확인은 `getUser` 네트워크 호출 대신 로컬 세션 기반으로 전환했고, 로그인/회원가입 경로는 미들웨어 인증 조회를 생략한다.
- 렌더링 중복을 줄이기 위해 인증 상태는 루트 레이아웃의 `AuthProvider`에서 한 번만 구독하고 하위 화면들이 공유한다.
- 로컬 개발 서버 지연 완화를 위해 기본 개발 번들러는 `webpack`으로 전환했고, `turbopack`은 선택 실행(`npm run dev:turbo`)으로 분리했다.
- Next 워크스페이스 루트는 프로젝트 디렉터리로 명시해 상위 폴더 감시에 따른 개발 서버 지연을 방지한다.
- 전역 `scroll-behavior: smooth`를 유지하는 대신 루트 `<html>`에 `data-scroll-behavior="smooth"`를 명시해 Next 라우트 전환 경고를 제거한다.
- 회원가입 성공 시 로그인 페이지로 이동하고, 로그인 성공 시 루트 페이지로 이동한다. 회원가입 실패 메시지는 한국어로 표시한다.
- 회원가입에서 비밀번호 확인이 일치하지 않으면 페이지 이동 없이 같은 화면에서 불일치 메시지를 보여줘 입력한 이름·이메일·비밀번호 값이 유지되도록 한다.
- 회원가입 오류 메시지는 배경 박스 없이 붉은 계열 텍스트만 표시해 입력 폼 흐름을 덜 끊도록 정리했다.
- 의약품 검색 카드에서도 내 복용약 등록/제거가 가능하고, 로그인/회원가입 화면에서는 홈으로 이동할 수 있다.
- 검색/상세 화면의 내 복용약 등록 버튼은 클릭 이벤트 전파를 차단해 현재 페이지에 머문 채 등록/제거되도록 유지한다.
- Stitch에 추가한 뒤로가기/홈/목록/필터 버튼은 공통 액션 버튼 컴포넌트로 옮겨서 검색/상세/내 복용약 상단에 아이콘+라벨형 UI로 적용한다.
- 사용자가 남겨둔 액션 버튼만 유지하도록 검색/내 복용약 상단 정렬을 정리했고, 헤더의 AYAK 아이콘은 `ayak_brand_logo` SVG 형태로 교체했다.
- `stitch_ayak_medication_manager`의 `아이콘.png`와 `파비콘.png`를 각각 공용 로고와 브라우저 아이콘 자산으로 연결했고, 브라우저 파비콘은 `app/icon.png`와 메타 아이콘 설정이 같은 이미지를 사용한다.
- 사용자가 교체한 최신 `아이콘.png`를 다시 동기화했고, 오래된 `app/favicon.ico`는 제거해 브라우저 파비콘이 `app/icon.png` 단일 경로만 사용하도록 정리했다.
- 파비콘이 너무 작아 보이던 문제를 줄이기 위해 가로형 `파비콘.png` 대신 정사각형 `아이콘.png`를 `app/icon.png`와 파비콘 자산에 다시 연결했다.
- 루트 `icon.svg`를 `app/icon.svg`로 복사하고, 브라우저 기본 아이콘/shortcut은 SVG를 우선 사용하며 `apple` 아이콘만 기존 PNG fallback으로 유지한다.
- 헤더와 로그인/회원가입 화면의 공용 로고 컴포넌트 `AyakBrandLogo`도 `icon.svg`를 사용해, 파비콘과 화면 내부 브랜드 아이콘 자산을 같은 소스로 맞춘다.
- 검색 페이지의 필터 버튼은 제목 위 단독 줄이 아니라 `의약품 검색` 헤더와 같은 줄의 보조 액션으로 배치해 상단 여백이 과하게 남지 않도록 정리했다.
- 내 복용약 페이지는 카드에서 효과 요약과 효능을 보여주고, 부작용은 별도 통합 섹션에서 모아 보여준다.
- 내 복용약 카드 레이아웃은 가로형으로 재구성했고, 부작용 모아보기는 약품별 가로 표 형태로 비교할 수 있게 구성했다.
- 의약품 검색 카드도 부작용/주의 중심이 아니라 효과 요약과 효능 중심으로 정보를 보여준다.
- 홈 화면은 검색 중복 입력 대신 서비스 소개, 핵심 기능, 내 복용약 요약, 빠른 진입 CTA 중심의 대시보드형 시작점으로 재구성했다.
- 홈의 빠른 진입 네비게이션은 제거했고, 우측 보조 영역은 복용약 요약과 추천 의약품 중심으로 정리했다. 의약품 검색 페이지는 태그형 필터로 효과/상황별 탐색을 지원한다.
- 홈 왼쪽의 `효과 확인 / 부작용 비교 / 내 복용약 관리` 설명 카드는 가로 분할 대신 세로 스택으로 배치해 히어로 하단 여백이 어색하지 않도록 정리했다.
- 홈 루트 페이지는 데스크톱에서 스크롤 없이 한 화면에 들어오도록 상하 패딩과 카드 간격을 줄이고, 핵심 기능 카드는 하단 3열 요약으로 압축했다.
- 홈 좌측 히어로는 CTA 아래 여백이 과하게 비지 않도록 상단 소개 영역 옆에 `Quick Guide` 보조 패널을 붙이고, 기능 카드와의 간격을 줄여 밀도를 다시 맞췄다.
- 홈 히어로 제목은 사용자가 정리한 4줄 카피를 유지하되 글자 크기와 줄간격, 본문 폭을 약간 줄여 더 단정하게 보이도록 다듬었다.
- 루트 홈 경로(`/`)는 `AppShell` 최상단을 `flex flex-col` 구조의 `h-dvh overflow-hidden`으로 고정해, 다른 페이지 스크롤에는 영향 없이 메인페이지만 브라우저 스크롤이 생기지 않도록 유지한다.
- 의약품 검색 카드는 효과 요약과 주성분을 보여주며, 카테고리 태그 필터는 버튼으로 숨기거나 펼칠 수 있다.
- 내 복용약 카드도 효과 요약과 주성분 기준으로 맞추고, 부작용 모아보기는 약별 나열 대신 부작용 키워드별 대응 가이드를 모아보는 구조로 유지한다.
- 내 복용약 화면에는 `통합 복용 주의` 섹션이 있고, 저장한 약 전체를 기준으로 식전·식후 복용, 운전 주의, 음주/병용 주의 같은 생활형 안내를 규칙 기반으로 묶어 보여준다.
- `통합 복용 주의`는 임신·수유, 병용 주의, 운전 주의처럼 먼저 봐야 할 안내가 위로 오도록 우선순위를 두고, 문구도 생활 안내형보다 더 의료 상담형 톤으로 다듬어 보여준다.
- `통합 복용 주의`와 `부작용 모아보기`는 기본적으로 상위 4개만 먼저 보여주고, 초과 항목은 `더보기` 버튼으로 펼쳐서 본다.
- `supabase/migrations/20260517_create_medications.sql`에 `medications` 테이블 스키마를 추가했고, `supabase/seeds/medications.sql`에 현재 더미 4건 seed를 준비했다.
- 앱의 홈, 검색, 상세, 내 복용약 화면은 현재 `medications` 테이블을 실제 조회 소스로 사용한다.
- `scripts/import-medications.mjs`는 공공데이터포털 `DrbEasyDrugInfoService`를 페이지 단위로 수집해 `medications` 테이블에 upsert한다.
- `supabase/migrations/20260517_enable_medications_public_read.sql`은 `medications`에 대해 `is_active = true` 행만 anon/authenticated가 조회할 수 있는 공개 읽기 정책을 정의한다.
- 적재 스크립트 실행에는 `SUPABASE_SERVICE_ROLE_KEY`, `PUBLIC_DATA_API_SERVICE_KEY` 환경변수가 필요하고, 주성분은 현재 API 응답에 없어 `ingredient`를 `null`로 적재한다.
- 적재 스크립트는 `PUBLIC_DATA_INGREDIENT_API_SERVICE_KEY`, `PUBLIC_DATA_INGREDIENT_API_URL`이 있으면 `DrugPrdtPrmsnInfoService07/getDrugPrdtPrmsnInq07`를 추가 호출해 `ITEM_SEQ` 기준으로 `ingredient`를 보강한다.
- `npm run import:ingredients`는 의약품 본 적재 없이 주성분 보강만 별도로 수행하는 경로다.
- 공공 의약품 자동 적재 스크립트는 실제 실행 검증을 마쳤고, 응답 총건수 4759 중 배치 내부 중복 `itemSeq`를 제거한 4742건이 적재됐다.
- 의약품 카드와 상세 상단의 이름 표기는 괄호 앞 대표 이름과 괄호 내부 보조 이름을 분리해 긴 상품명 줄바꿈이 덜 어색하게 보이도록 조정했다.
- 검색 카드 헤더는 번호형 접두를 표시 단계에서 제거하고, 상단 `효과` 배지 높이와 헤더 최소 높이를 고정해 카드 UI 흔들림을 줄였다.
- 검색 카드의 긴 의약품명은 카드 밖으로 넘치지 않도록 강제 줄바꿈하고, 효과 요약은 3줄까지만 노출한 뒤 말줄임 처리한다.
- 내 복용약 카드도 긴 의약품명이 넘치지 않도록 강제 줄바꿈하고, 효과 요약은 3줄 말줄임으로 제한한다.
- 검색 페이지는 `q`, `tag`, `page` 기준 서버 조회로 동작하며, `medications`를 20개 단위로 페이징해 전건 클라이언트 로딩을 피한다.
- 검색 실행 시 태그 필터는 자동으로 `#전체`로 초기화해 이름 검색 결과가 이전 태그 선택 때문에 숨겨지지 않도록 한다.
- 상세 페이지 라우팅은 공공 API 기반 긴 `slug` 인코딩 이슈를 줄이기 위해 카드 링크를 `id` 기준으로 이동시키고, 상세 조회는 `id`와 기존 `slug`를 모두 허용한다.
- 의약품 상세 페이지는 검색 복귀 링크, 핵심 정보 요약 카드, 섹션 바로가기, 빈 데이터 fallback 문구를 추가해 읽는 순서를 단순화했다.
- 의약품 상세 페이지의 주의사항은 태그를 제거하고 `사용 방법` 하위의 번호형 리스트로 재배치해, 복용 방법 다음에 바로 이어서 읽을 수 있게 정리했다.
- 의약품 상세 페이지의 부작용은 별도 우측 컬럼 대신 본문 맨 아래에 배치해, `효능 → 사용 방법 → 주의사항 → 부작용` 순서로 순차적으로 읽을 수 있게 정리했다.
- 상세/내 복용약의 부작용 표기는 원문 문장 대신 핵심 키워드 칩 중심으로 축약하고, `부작용이 발생할 경우 사용을 즉각 중단하고 의사 또는 약사와 상의하십시오.` 안내 문구는 한 번만 통합 노출한다.
- 내 복용약의 부작용 모아보기는 부작용 키워드별로 관련 약 수와 대응 가이드를 함께 보여주며, 예를 들어 `졸림`, `어지러움`에는 운전·기계 조작 주의 문구를 연결한다.
- `useUserMedications`는 클라이언트 공유 스토어로 동작해 약 삭제/추가 시 내 복용약, 마이페이지 복용약 현황, 홈 복용약 요약이 같은 탭에서 즉시 함께 갱신된다.
- 복용약 등록은 `user_id + medication_id` 기준 idempotent 하게 처리해, 이미 저장된 약이라도 클라이언트 상태가 늦게 따라온 경우 중복 오류 대신 재동기화되도록 유지한다.
- 검색/상세의 내 복용약 버튼은 사용자 복용약 목록 로딩 전에는 비활성화해, 초기 동기화 전 저장 상태 오판으로 인한 중복 등록 시도를 줄인다.
- 홈/마이페이지/내 복용약은 현재 `medicationIds` 기준으로 공개 조회 가능한 `medications`를 브라우저에서 다시 읽어, 저장 직후 목록/개수가 서버 초기 props와 어긋나지 않도록 동기화한다.
- 내 복용약 화면에는 브라우저 확인창을 거치는 `전체 삭제` 버튼이 있고, 확인 시 현재 사용자 `user_medications`를 일괄 삭제한다.
- 주성분 표시는 공통 정제 함수를 거쳐 괄호 찌꺼기와 불필요한 끝 구두점을 줄이고, 검색/상세/내 복용약 카드에서는 강제 줄바꿈으로 overflow를 방지한다.
- 마이페이지의 건강정보는 회원 기본정보와 분리된 `복약 안전 프로필` 도메인으로 정의하고, `기본 상태/생활 습관/기저 질환/약물 안전` 4개 묶음으로 다룬다.
- 건강정보 1차 항목은 나이, 임신 여부, 수유 여부, 음주 빈도, 운전 여부, 간 질환 여부, 신장 질환 여부, 고혈압 여부, 천식 여부, 위장 질환 여부, 알레르기, 현재 복용 중인 약이다.
- 나이는 직접 입력하고, 화면 표시와 후속 경고 규칙에서는 `0-12세 어린이`, `13-18세 청소년`, `19-58세 성인`, `59세 이상 고령자` 기준으로 자동 분류한다.
- 음주 정보는 `거의 안 마심`, `월 1~3회`, `주 1~2회`, `주 3회 이상` 선택형 빈도로 입력한다.
- 건강정보는 마이페이지에서 입력·수정하되, 실제 활용은 의약품 검색/상세/내 복용약의 주의 안내 강화에 둔다.
- 내 복용약의 `통합 복용 주의`는 건강 프로필과 맞는 항목에 우선순위 가산점을 적용해, 임신/수유, 운전, 음주, 간/신장 질환, 고혈압, 천식, 위장 질환, 알레르기, 병용약 관련 안내가 위로 올라올 수 있게 정렬한다.
- 의약품 상세 상단에는 현재 약 하나를 기준으로 건강 프로필과 직접 맞닿는 `맞춤 주의` 카드만 따로 보여줘, 사용 방법과 전체 주의사항을 읽기 전에 먼저 확인할 수 있게 한다.
- 건강정보 1차 저장은 별도 Supabase 스키마 없이 로그인 사용자별 브라우저 `localStorage`로 관리한다.
- 다음 핵심 범위는 공공 API 응답 필드의 최종 매핑 규칙 보정, 주성분 보강 데이터 소스 검토, 적재 운영 절차 정리다.
